# Strategic Automation Map

## What to Do

Identify which of your optimised processes are ready for automation. Map every repeatable task, score it for automation potential, and prioritise based on time savings and implementation feasibility. The golden rule: never automate a broken process. Fix it first, then automate it. Automating waste just produces waste faster.

## Why It Matters

Automation is one of the most powerful tools available to small businesses today. Tasks that once required a full-time employee can now be handled by software, AI, or simple workflow tools. But automation done wrong creates more problems than it solves. Automating a bad process locks in the inefficiency. Automating before you understand the process means you cannot troubleshoot when things go wrong. The businesses that win with automation are the ones that optimise first (using the earlier actions in this playbook) and automate second.

## How to Use This

Inventory your automatable tasks, score them, and build a phased automation plan.

---

## Inventory Automatable Tasks

- [ ] **Review your optimised processes from Process Archaeology** - Focus on the processes you have already simplified and standardised. These are the candidates for automation.
- [ ] **List every task that is: repetitive, rule-based, and high-volume** - If a human does the same thing the same way more than 10 times per week, it is a candidate.
- [ ] ⚡ Quick Win: Identify one task you do daily that follows the same steps every time. This is your easiest automation candidate. Common examples: sending follow-up emails, updating a spreadsheet, scheduling appointments, generating invoices.
- [ ] **Categorise tasks by type:**
  - Data entry and transfer (moving information between systems)
  - Communication (sending standard emails, reminders, confirmations)
  - Scheduling (booking meetings, allocating resources)
  - Reporting (generating regular reports from existing data)
  - Document creation (proposals, contracts, invoices from templates)
  - Monitoring and alerts (flagging when a metric hits a threshold)

## Score Each Task for Automation

- [ ] **Time spent** - How many hours per week does this task consume across the team? (1 = less than 30 min, 5 = 5+ hours)
- [ ] **Consistency** - How standardised is the task? Does it follow the same steps every time? (1 = highly variable, 5 = identical every time)
- [ ] **Error rate** - How often does the manual version produce mistakes? (1 = rarely, 5 = frequently)
- [ ] **Integration complexity** - How many systems need to connect to automate this? (1 = 3+ systems, 5 = single system or simple connection)
- [ ] **Total score per task** - Higher scores = higher automation priority. Focus on tasks that are time-consuming, consistent, error-prone, and simple to connect.

## Build Your Automation Plan

- [ ] **Phase 1 (Next 30 days): Quick automations** - Tasks scoring 15+ that can be automated with tools you already have (email rules, CRM workflows, spreadsheet formulas, Zapier). No new software needed.
- [ ] **Phase 2 (30-90 days): Standard automations** - Tasks scoring 12+ that need a new tool or integration. Research options, choose a tool, and implement.
- [ ] **Phase 3 (90+ days): Advanced automations** - Tasks requiring more complex setup, AI tools, or custom development. Plan these but do not start until Phases 1 and 2 are stable.
- [ ] **For each automation, define: the manual process being replaced, the tool or method, the owner responsible for setup, and the success metric (time saved, errors reduced, or speed improved)**
- [ ] **Build a "human check" into critical automations** - Automation should handle the routine. Humans should review the exceptions. Never fully automate something where an error has serious consequences without a checkpoint.
- [ ] **Set a review after 60 days** - Is each automation working as expected? Are there new manual tasks that have emerged and need automation?

## Expected Outcome

- A complete inventory of automatable tasks with scores
- Clear automation priorities based on time savings and feasibility
- A phased plan (30/60/90 days) with specific automations for each phase
- Quick wins implemented in the first 30 days
- Human checkpoints built into critical automations
- A review cycle to assess automation performance and add new candidates

## Coach Notes

The biggest mistake is jumping to tools before understanding the process. Clients hear "automation" and immediately want to buy software. Slow them down. The first question is "what exactly are we automating and is the current process worth automating?" If the answer is no, go back to the optimisation actions first.

Start with the boring automations. Data entry, email follow-ups, and appointment scheduling are not exciting, but they save real hours every week. The complex, impressive automations can wait until the basics are solid.

Watch for over-automation. Some client interactions benefit from a human touch. A personalised welcome email from a real person is worth more than a perfectly timed automated sequence. Help clients draw the line between tasks that should be automated and moments that should stay human.
