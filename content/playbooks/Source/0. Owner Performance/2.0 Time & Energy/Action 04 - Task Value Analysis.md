# Task Value Analysis

## What to Do

Take every recurring task on your plate and categorise it by value (high, medium, low), then apply the E-A-T framework: Eliminate, Automate, or Transfer. The goal is to identify what should come off your plate entirely and create a concrete plan for making that happen.

## Why It Matters

You already know you are doing work beneath your pay grade. The problem is it all feels necessary in the moment. This exercise forces you to evaluate each task objectively and make a decision about its future. Without this analysis, delegation stays theoretical. With it, you have a specific list of tasks, each with a clear action: kill it, automate it, or hand it to someone else.

## How to Use This

Pull your task list from the Time Scan Tracker (Action 2) or 15-Minute Time Tracker (Action 3). If you have not done those yet, list every recurring task you can think of from the past month. Then work through the framework below.

---

## Task Inventory

- [ ] List every recurring task you perform weekly or monthly (aim for 20-30 items minimum)
- [ ] For each task, note approximately how many hours per week or month it takes
- [ ] Categorise each task: **High Value** (only you can do this, directly drives revenue or strategy), **Medium Value** (you are good at it but someone else could learn), **Low Value** (anyone could do this with basic training)
- [ ] Total the hours in each category - what percentage of your time goes to low and medium value work?

## The E-A-T Framework

### Eliminate

- [ ] Review all Low Value tasks - which ones would have zero impact if you simply stopped doing them?
- [ ] ⚡ Quick Win: Pick one task you can stop doing this week with no consequences. Stop doing it.
- [ ] Identify tasks that exist because "we have always done it this way" rather than because they produce results
- [ ] Flag any reporting or admin that nobody reads or uses

### Automate

- [ ] Review remaining Low and Medium Value tasks - which ones follow a repeatable pattern?
- [ ] Identify tasks that involve moving data between systems, sending routine communications, or creating standard documents
- [ ] For each automation candidate, note the tool or method that could handle it (templates, software, Zapier, AI tools)
- [ ] Prioritise automations by time saved vs. effort to set up

### Transfer (Delegate)

- [ ] Review remaining Medium Value tasks - who on your team could handle these with training?
- [ ] For each delegation candidate, identify the specific person and what training they would need
- [ ] Write a simple brief for each task to be delegated: what it is, how to do it, what good looks like, when it is due
- [ ] Set a 2-week target to hand off at least one task with proper documentation
- [ ] For tasks where no current team member fits, note whether this is a future hire need or a contractor/freelancer task

## Action Plan

- [ ] Create a simple table: Task | Category | E-A-T Decision | Who/How | By When
- [ ] Set a target: move at least 5 hours per week of low/medium value work off your plate within 30 days
- [ ] Schedule a review in 30 days to check progress and identify the next batch

---

## Expected Outcome

- Every recurring task categorised by value with a clear action assigned
- An immediate short list of tasks to eliminate this week
- A prioritised automation list with specific tools identified
- A delegation plan with names, training needs, and deadlines
- A measurable target for hours reclaimed over the next 30 days

## Coach Notes

This works best when combined with data from the time tracking exercises (Actions 2 and 3). Without that data, clients tend to underestimate how much time goes to low-value work. The biggest resistance point is delegation: clients will say "it is faster to do it myself" or "no one else can do it as well." Challenge both. Faster to do it yourself today means you are still doing it in a year. Good enough from someone else beats perfect from you if it frees you for higher-value work. Focus on getting the first delegation done successfully to build confidence.
